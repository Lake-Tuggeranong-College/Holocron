# STRIDE Threat Model

## Cybersecurity for IoT Systems

---

## What is STRIDE?

- Structured threat modeling framework.
- Identifies potential system vulnerabilities.
- Developed originally by Microsoft.
- Organises threats into six categories.
- Focuses on proactive mitigation.
- Secures network data pathways.

---

## The STRIDE Mnemonic

- **S**poofing identity.
    
- **T**ampering with data.
    
- **R**epudiation.
    
- **I**nformation disclosure.
    
- **D**enial of service.
    
- **E**levation of privilege.

---

## Spoofing Identity

- Impersonating trusted network users.
    
- Bypassing target host authentication.
    
- Accessing systems using stolen keys.
    
- Creating rogue device connections.
    
- Violates critical system trust.
    
- Mitigation: Implement secure authentication.
    
---
## Tampering with Data

- Unauthorized modification of datasets.
    
- Altering database network tables.
    
- Intercepting active telemetry packets.
    
- Violates data transmission integrity.
    
- Destroys raw sensor validity.
    
- Mitigation: Enforce data signatures.
    
---
## Repudiation Gaps

- Users denying executed actions.
    
- Missing or altered audit trails.
    
- Bypassing individual accountability metrics.
    
- Untraceable system database modifications.
    
- Confuses legal liability structures.
    
- Mitigation: Maintain secure logging.
    
---
## Information Disclosure

- Leaking private application database secrets.
    
- Transmitting database packets unencrypted.
    
- Reading plaintext credentials directly.
    
- Violates user privacy laws.
    
- Exposes critical system architectures.
    
- Mitigation: Enforce TLS/SSL encryption.
    
---
## Denial of Service

- Flooding target network ports.
    
- Overwhelming system host memory.
    
- Crashing central database connections.
    
- Blocks legitimate user utility.
    
- Disrupts real-time telemetry pipelines.
    
- Mitigation: Restrict interface rates.
    
---
## Elevation of Privilege

- Gaining unauthorized administrator rights.
    
- Exploiting unpatched system software.
    
- Escaping isolated application containers.
    
- Compromises absolute system control.
    
- Accessing restricted internal resources.
    
- Mitigation: Apply least privilege roles.
    
---
## Security Goal Alignment

- STRIDE maps to security goals.
    
- Spoofing violates user authentication.
    
- Tampering violates data integrity.
    
- Repudiation violates system non-repudiation.
    
- Disclosure violates data confidentiality.
    
- Denial violates resource availability.
    
---
## Modeling Methodology

- Decompose system network architecture.
    
- Map all target interfaces.
    
- Identify vulnerable data routes.
    
- Categorize threats using STRIDE.
    
- Rank vulnerabilities by danger.
    
- Implement defensive software controls.
    
---
## IoT Security Mitigations

- Secure ESP32 API tokens.
    
- Use isolated Docker subnets.
    
- Restrict database access privileges.
    
- Encrypt data in transit.
    
- Routinely capture network traffic.
    
- Audit logs for anomalies.
    
---
## Security Philosophy

- "Security is a process..."
    
- "...not a standalone product."
    
- Continuous iteration is critical.
    
- Address security during design.
    
- Keep updating defense layers.
    
- Always verify system integrity.
