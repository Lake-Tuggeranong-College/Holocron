# What is SPIFFS?

> [!note] Generated by AI, verified by human.

SPIFFS stands for **Serial Peripheral Interface Flash File System**. It's a file system specifically designed for storing data on the internal flash memory of ESP32 microcontrollers.  It's *not* a general-purpose file system like you'd find on a computer.  It's optimised for the ESP32's flash memory and the way it's accessed.

**How Does it Work?**

1. **Flash Memory:** The ESP32 has a relatively small amount of internal flash memory (typically 4MB or 8MB, depending on the specific board). This is where SPIFFS stores your files.

2. **File System Structure:** SPIFFS organises files within a hierarchical directory structure, just like a traditional file system.  However, it's *not* a full-blown filesystem like NTFS or FAT32. It's a simplified, optimised structure.  Here's a simplified breakdown:
   * **Root Directory:** The top-level directory.
   * **Subdirectories:**  You can create subdirectories to organise your files.
   * **Files:**  These are the actual data you store (HTML, CSS, JavaScript, images, etc.).

3. **SPI Interface:**  SPIFFS uses the Serial Peripheral Interface (SPI) to communicate with the ESP32's flash memory. The SPI interface is a hardware protocol that allows the ESP32 to quickly read and write data to the flash.

4. **Libraries & APIs:** The ESP32 framework provides libraries (like `LittleFS`) that handle the complexities of interacting with SPIFFS.  You don't typically deal with the low-level SPI communication directly.  The library takes care of:
   * **File Creation/Deletion:**  Creating new files, deleting existing ones.
   * **File Reading/Writing:**  Reading data from files and writing data to files.
   * **Directory Navigation:**  Moving between directories.
   * **Error Handling:**  Dealing with potential errors during file operations.

**Key Features & Optimisations of SPIFFS:**

* **Small Footprint:**  SPIFFS is designed to minimise the overhead of the file system itself. This is crucial because flash memory has limited write cycles and a relatively slow write speed.
* **Fast Access:**  It's optimised for fast read and write access to files, leveraging the ESP32's SPI interface.
* **Automatic Garbage Collection:**  SPIFFS automatically manages the flash memory by identifying and removing unused blocks of data. This helps to extend the lifespan of the flash memory and improve performance.
* **Atomic Operations:**  SPIFFS uses atomic operations to ensure that file operations are completed reliably, even in the presence of interruptions.

**Benefits for a Web-Enabled Project**

This is where SPIFFS shines for ESP32 web projects:

* **Static Content Storage:**  A web server running on the ESP32 typically serves *static* content (HTML, CSS, JavaScript, images) to a user's browser. SPIFFS provides a convenient way to store this content directly on the ESP32's flash memory.
* **No External Storage Needed:** You don't need an SD card or other external storage device. This simplifies the hardware design and reduces the cost.
* **Offline Functionality:**  The ESP32 can serve content even when it's not connected to the internet.  This is useful for creating local web servers or for providing offline access to data.
* **Easy Updates:**  You can update the content on the ESP32 without needing to reprogram the entire firmware.  Just overwrite the files in the SPIFFS directory.
* **Reduced Bandwidth Usage:** Serving content directly from the ESP32's flash memory reduces the amount of data that needs to be downloaded from a remote server.

**Example Scenario:**

Imagine you're building a web interface for a sensor that's collecting data.  You could:

1. Store the HTML, CSS, and JavaScript for the web interface in SPIFFS.
2. Store the sensor data (e.g., as JSON files) in SPIFFS.
3. The ESP32's web server would serve these files directly from SPIFFS to the user's browser.

**Important Considerations:**

* **Flash Memory Endurance:** Flash memory has a limited number of write cycles. Frequent writing to SPIFFS can reduce the lifespan of the flash memory.  Optimise your code to minimise unnecessary writes.
* **File Size Limits:**  The size of the files you can store in SPIFFS is limited by the available flash memory.
* **LittleFS (Recommended):**  The `LittleFS` library is the modern and recommended way to work with SPIFFS on the ESP32. It's more robust and offers better performance than the older SPIFFS library.

# Implementing SPIFFS in Platform IO

> [!note] Important - **do not install** the SPIFFS plugin referenced on other sites. 
> PlatformIO has the ability to interact with the SPIFFS partition without the need for the plugin.

## Create HTML file to upload

First, in VSCode open the project, and create a `data` folder in the root directory of the project.

![[_Robotics and Mechatronics/1 - Building and Programming Circuits/_projects/Smart House/_images/spiffsDataFolder.png]]

Create a file in the `data` folder called `index.html` to test the process. Place the following code in the file.

![[spiffsIndex.png]]

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <title>ESP32 SPIFFS Demo</title>
 
</head>
<body>
    <h1>ESP32 SPIFFS Demo</h1>
    <p>
      This page demonstrates loading a file from SPIFFS on the ESP32.
    </p>

</body>
</html>

```

## Upload to ESP32

Click the PlatformIO icon on the left of the VSCode window. Then expand esp32dev (or whatever board you've got), then click **Build FilesystemImage**.

![[spiffsUploadFileSystem.png]]

Then you'll see the terminal show its uploading the data. It looks similar to uploading code to the board, with some slight differences.

# SPIFFS Code

Now that the file/s has been uploaded to the board, you can update your project to load the file to server to the browser.

Add the following code to the top of your `main.cpp`.

```arduino
#define FORMAT_SPIFFS_IF_FAILED true
#include "SPIFFS.h"
#include <ESPAsyncWebServer.h>
AsyncWebServer server(80);
```

Add the following to your `setup()` function:

```arduino
  if (!SPIFFS.begin(FORMAT_SPIFFS_IF_FAILED)) {
    // Follow instructions in README and install
    // https://github.com/me-no-dev/arduino-esp32fs-plugin
    Serial.println("SPIFFS Mount Failed");
    return;
  }
```

Update/replace your webserver routes to this format:

```arduino
//Configure web routes
server.on("/", HTTP_GET, [](AsyncWebServerRequest * request) {
	request->send(SPIFFS, "/index.html", "text/html");
});
```

# Including Telemetry data in HTML

If you want to output values into the HTML, such as the temperature value or another sensor, you can do this through a number of steps.

1. Include a placeholder string in place that you want to replace later, such as `%TEMPERATUREVALUE%`. E.g.
```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <title>ESP32 SPIFFS Demo</title>
 
</head>
<body>
    <h1>ESP32 SPIFFS Demo</h1>
    <p>
      This page demonstrates loading a file from SPIFFS on the ESP32.
    </p>
    <p> The temperature is currently: %TEMPERATUREVALUE%</p>

</body>
</html>
```
2. Include a `processor()` function which detects the placeholder string and replaces it with the value you want.
```arduino
String processor(const String& var) {
  Serial.println(var);
  
  if (var == "TEMPERATUREVALUE") {
    return String(dhtTemperature);
  }
  if (var == "EXAMPLEOUTPUT") {
    // Process data...
    return "This is example data";
  }
  
  return String();
}
```

> [!note] The data needs to be a string when returned.
3. Include/update the route to run the processor function prior to sending the html to the browser.
```arduino
  server.on("/temperature", HTTP_GET, [](AsyncWebServerRequest * request) {
    request->send(SPIFFS, "/temperature.html", "text/html", false, processor);
  });
```

> [!note] The `false` parameter in `request->send()` means the file will be displayed in the browser. Change this to true if you want to force the file to download (e.g., for log files).
